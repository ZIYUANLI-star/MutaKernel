# 断电续跑指南

> 实验中断后，替换一个文件，杀掉旧进程，重跑同一命令即可接着跑。

---

## 一、需要发给师弟的文件

只需要发 **1 个文件**：

| 文件（本地路径） | 服务器上替换位置 |
|-----------------|-----------------|
| `MutaKernel/scripts/_stress_worker.py` | `/home/kbuser/projects/MutaKernel/scripts/_stress_worker.py` |

这个文件包含了编译缓存优化，替换后同一 kernel 不再重复编译 CUDA 扩展，整体速度提升约 2-3 倍。

**其余文件完全不需要动。已有的结果数据、checkpoint 都不受影响。**

---

## 二、师弟操作步骤（复制粘贴即可）

```bash
# ===== 第 1 步：停掉正在跑的进程 =====
ps aux | grep run_fullscale_diff_test | grep -v grep
# 记下进程号（第二列的数字），然后 kill 掉
kill <进程号>

# 如果用了 tmux，先 attach 进去再 Ctrl+C：
# tmux attach -t exp
# 然后 Ctrl+C 停止

# ===== 第 2 步：替换文件 =====
# 把我发给你的 _stress_worker.py 上传到服务器，覆盖旧文件
# 方式 A：scp 上传
scp _stress_worker.py kbuser@服务器IP:/home/kbuser/projects/MutaKernel/scripts/_stress_worker.py

# 方式 B：如果已经在服务器上了，直接 cp
cp /tmp/_stress_worker.py /home/kbuser/projects/MutaKernel/scripts/_stress_worker.py

# ===== 第 3 步：重跑（自动接着上次断的地方继续）=====
cd /home/kbuser/projects/MutaKernel
source /home/kbuser/projects/KernelBench-0/.venv/bin/activate

# 用 tmux 防止 SSH 断连
tmux new -s exp
python scripts/run_fullscale_diff_test.py --dataset all

# 断开 tmux：按 Ctrl+B，然后按 D
# 重连查看：tmux attach -t exp
```

---

## 三、为什么可以直接接着跑

脚本有 **checkpoint 机制**：
- 每跑完一个 kernel，结果立刻写入 `第三次实验汇总/results/{数据集}/checkpoint.json`
- 重新运行时，脚本读取 checkpoint，自动跳过已完成的 kernel
- 不需要删除任何结果文件，不需要加任何参数

## 四、查看进度

```bash
# 看当前跑到第几个
python -c "
import json
from pathlib import Path
for ds in ['cuda_l1', 'ai_cuda_engineer', 'tritonbench_g', 'apex_new']:
    cp = Path(f'第三次实验汇总/results/{ds}/checkpoint.json')
    if cp.exists():
        data = json.load(open(cp))
        print(f'  {ds}: 已完成 {len(data)} 个 kernel')
    else:
        print(f'  {ds}: 尚未开始')
"
```

## 五、跑完的标志

日志中出现 `ALL DATASETS DONE` 就是跑完了。

或者检查每个数据集目录下是否生成了 `summary.json`：

```bash
ls 第三次实验汇总/results/*/summary.json
```

---

## 六、注意事项

- **不要删除** `第三次实验汇总/results/` 下的任何文件（checkpoint 和 details 都是已有成果）
- 替换的这个文件只影响运行速度，不影响测试结果的正确性
- 如果进程卡住超过 30 分钟没有新输出，直接 kill 重跑就行
