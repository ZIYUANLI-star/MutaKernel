## 5. 逐个变异体详细分析

> 以下按内核分组列出每个未杀死变异体的变异内容、Tier 分类、存活原因类别及 LLM 给出的完整存活原因分析。

### L1_P1 (1 个未杀死)

#### #1 `L1_P1__relop_replace__2`

- **算子类型**: 关系运算替换 (`relop_replace`)
- **原始片段**: `<`
- **Tier**: 2
- **原因类别**: 谓词不可达 (`predicate_unreachable`)
- **存活原因分析**:

  The mutation replaces '<' with '<=' in the boundary check for loading tile A. However, with the fixed shape N=2048 which is a multiple of TILE_SIZE=32, the maximum column index accessed is 2047, which is strictly less than N. Both conditions evaluate identically (true) for all threads. The else branch (set to 0.0f) is never taken, so the computation is bit-identical. All tested inputs, including extreme values and boundary stress, cannot trigger a difference because the condition never changes from false to true or vice versa. The mutant is therefore unkillable under the fixed-shape contract.

### L1_P2 (1 个未杀死)

#### #2 `L1_P2__relop_replace__1`

- **算子类型**: 关系运算替换 (`relop_replace`)
- **原始片段**: `<`
- **Tier**: 1
- **原因类别**: 谓词不可达 (`predicate_unreachable`)
- **存活原因分析**:

  The mutated condition `A_row <= M` is always true just like the original `A_row < M` because the maximum row index (1023) is always less than M (1024). Since the grid exactly covers M rows with block size 16, no thread ever accesses a row index equal to M. Therefore both conditions yield identical results for every possible input tensor value. The mutation is functionally equivalent and cannot be distinguished under the fixed-shape contract.

### L1_P8 (2 个未杀死)

#### #4 `L1_P8__mask_boundary__0`

- **算子类型**: 边界条件变异 (`mask_boundary`)
- **原始片段**: `global_row < M`
- **Tier**: 2
- **原因类别**: 谓词不可达 (`predicate_unreachable`)
- **存活原因分析**:

  The mutation tightens the boundary guard from `global_row < M` to `global_row < M-1` in the A-tile loading loop. However, the affected row (global_row = M-1 = 8204) is never used in the computation because the kernel only computes a fixed subset of rows per block (4 rows per block, rows 0-3 of each tile). The tile size TILE_M=128, but the row_in_tile calculation restricts coverage to only rows 0-3 within each tile. Therefore, for M=8205, rows 8196 through 8204 are never written, and the row 8204 is never loaded into a computation path. Hence, the mutation has no impact on the kernel's output, making it behaviorally equivalent to the original.

#### #5 `L1_P8__relop_replace__7`

- **算子类型**: 关系运算替换 (`relop_replace`)
- **原始片段**: `<`
- **Tier**: 2
- **原因类别**: 值不敏感 (`value_insensitive`)
- **存活原因分析**:

  The mutant survived because the out-of-bounds read introduced by the `<=` operator accesses memory that is not part of any input tensor and whose value is not determined by the contract's variable-value specification. Under the fixed-shape contract, we can only vary the values within the tensor shapes, not the memory layout or contents of adjacent allocations. Since the kernel always receives contiguous copies via `.contiguous().cuda()`, we cannot control what lies immediately after B. All tested value distributions resulted in that memory being zero (or indistinguishable from zero), making the mutant's output bitwise identical to the original. The kill strategy suggested in Layer 3 (placing B in a larger buffer with distinctive tail) fails because contiguous() would copy only the slice, discarding the extra data. Therefore, no value-level test can force a difference.

## 6. 关键发现与规律总结

### 6.1 谓词不可达（predicate_unreachable，56 个，45.5%）

这是最主要的存活原因。典型模式包括：

- **维度对齐**: 固定输入形状恰好是 block size / tile size 的整数倍，使得 `>=` 与 `>` 或 `<` 与 `<=` 的边界条件变异永远不被触发（如 L1_P1, L1_P2, L1_P5, L1_P9 等的 relop_replace）
- **死代码路径**: 内核包含多个特化路径（如简单/优化两套 kernel），但固定参数永远选择其中一条路径，使另一路径中的变异无效（如 L1_P12 的 8 个变异体全部在未使用的 simple kernel 中）
- **半精度分支**: kernel 包含 float16 特化分支，但固定输入为 float32，half kernel 从未被调用（如 L1_P88 的 3 个变异体）
- **affine 分支**: InstanceNorm 等内核包含 affine/non-affine 两条路径，固定模型总使用 affine=True（如 L1_P34 的 6 个变异体在 non-affine kernel 中）

### 6.2 值不敏感（value_insensitive，35 个，28.5%）

变异修改了代码但在固定配置下计算结果不变：

- **启动配置过量**: 将 grid size 计算从除法改为乘法，产生过多 block，但边界检查阻止了越界写入（如 L1_P9, L1_P16, L1_P27, L1_P91）
- **整数算术等价**: 特定维度值下 `*` 和 `/` 产生相同结果（如 inner_size=1 时 `d*stride` ≡ `d/stride`，L1_P90 的多个变异体）
- **共享内存对齐**: 共享内存数组大小从 256 改为 255，但 GPU 按 128 字节粒度分配，实际分配相同（如 L1_P48）
- **冗余同步移除**: `__syncthreads()` 在特定执行模式下是冗余的（如 warp 内天然同步、计算极短等，L1_P34, L1_P49, L2_P66）

### 6.3 路径未触发（path_not_triggered，21 个，17.1%）

变异代码存在于特定路径但测试未覆盖：

- **reduction dim 未覆盖**: 内核支持多个 reduction 维度，但测试仅使用默认 dim（如 L1_P49 的 dim2 kernel、L1_P52/L1_P53 的 dim0 kernel）
- **训练模式**: BatchNorm 训练分支的变异（如 L1_P33 的 3 个变异体）因测试跳过训练模式而存活
- **极端值域未命中**: 初始化值变异（如 `-INFINITY` → `-1e10f`）需要输入全部 < -1e10 才能暴露，值域压力未能覆盖（如 L1_P41, L1_P49）
- **GPU 非确定性**: `sync_remove` 变异引入竞态条件但在测试环境中未实际触发（如 L1_P100, L1_P94）

### 6.4 感染未传播（infection_no_propagation，10 个，8.1%）

变异确实改变了中间值，但未影响最终输出：

- **输出为常数**: kernel 输出硬编码为 1.0，内部计算的差异不传播（如 L2_P66 的 3 个变异体）
- **覆盖写**: 越界写入的值被后续正确的 block 覆盖（如 L1_P34, L1_P89）
- **统计量不影响输出**: running_stats 更新变异不影响当前前向传播输出（如 L1_P33）
- **group_size=1**: GroupNorm 中 group_size=1 导致归一化值恒为零（L2_P41）

### 6.5 需要配置变更（requires_config_change，1 个，0.8%）

L1_P89 的 `sync_remove__4`: 原始代码本身已存在竞态条件 bug，变异在已有 bug 的基础上修改无法被区分。
