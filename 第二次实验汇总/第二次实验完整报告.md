# MutaKernel 第二次实验完整报告

> **实验时间**: 2026 年 4 月  
> **实验对象**: KernelBench 基准中的 CUDA C++ GPU 内核  
> **实验框架**: MutaKernel V2.0（两阶段流水线架构）  
> **数据来源**:
> - Phase 1: `full_block12_results/summary.json`（及 `full_block12_results/details/*.json`）
> - Phase 2: `stress_enhance_results/details/*.json`（534 个文件，每个文件对应一个增强测试变异体）

---

## 1. 实验概览

本次实验采用 MutaKernel V2.0 的两阶段流水线架构，对 90 个 CUDA GPU 内核进行了完整的变异测试：

- **Phase 1**: 变异生成 + 四层等价变异体检测（EMD）流水线
- **Phase 2**: 融合增强测试与 LLM 迭代分析（对 Phase 1 中所有非 Killed/Stillborn/Strict Equivalent 变异体进行增强测试）

---

### 2.2 四层 EMD 流水线结果

| 层级 | 功能 | 检出数 | 输出状态 |
|------|------|--------|----------|
| Layer 0 | CUDA-aware 源码规范化 | 2 个 Strict Equivalent（CUDA+Python 文本完全相同） | STRICT_EQUIVALENT |
| Layer 1 | 算子特定静态等价规则 (4 条规则) | 8 个 Strict Equivalent（dead_host_constant 等规则命中） | STRICT_EQUIVALENT |
| Layer 2 | 动态 bitwise 检测 (112 轮 random+stress) | 264 个 Candidate Equivalent | CANDIDATE_EQUIVALENT 或保持 SURVIVED |
| Layer 3 | LLM 等价性验证 (DeepSeek) | 审查所有 STRICT 和 CANDIDATE，可推翻为 SURVIVED | 确认或推翻 |

> **说明**: EMD 流水线不产出 KILLED 状态。Layer 2 使用 bitwise 比较（比 allclose 严格），找到差异只意味着"不等价"，不等于 Phase 1/Phase 2 的 allclose 容差下能被杀死。详见 `docs/EquivalentDetection_V2_Plan.md` §4.3。
